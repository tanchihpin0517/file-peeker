# File Peeker

File Peeker is a planned Finder-like file browser written in Rust. Its
filesystem logic can run locally at first and on another machine in the future.

This repository currently contains **documentation only**. There is no working
application or Rust workspace yet.

## Design

```text
Rust TUI  ─┐
           ├─> Shared client library -> Host process -> Filesystem
Swift UI ──┘
```

- The **host** runs on the machine being browsed and performs filesystem work.
- The **client** starts and connects to the host and exposes a UI-independent
  asynchronous file-browser interface.
- A **UI** handles input and display only. The first UI uses Rust and Ratatui; a
  native macOS UI written in Swift is planned next.

Separating these components allows the Rust TUI and Swift UI to share the same
client implementation and host behavior. Rust uses the client's native API;
Swift uses bindings generated by UniFFI. The native macOS project is generated
and managed with XcodeGen. The host protocol and process lifecycle remain
private to the client. The same boundary also allows a future remote connection
without moving filesystem logic into each UI.

The client produces filesystem results asynchronously but does not update or
render either UI. Ratatui forwards results into its application event loop;
SwiftUI applies them to observable state on the main actor.

Each `BrowserClient` owns one dedicated host process. A long-lived control
connection defines their shared lifetime, while each filesystem operation uses
its own short-lived connection. This permits independent operations without
request IDs or response routing on one socket.

## First version

V1 is a local, macOS-only, read-only terminal browser. It will:

- Start in a supplied path or the current directory.
- Stream and display the direct children of that directory.
- Include hidden entries and keep filesystem enumeration order.
- Allow entering child directories and directory symlinks.

V1 will not include parent navigation, sorting, previews, search, filesystem
changes, remote connections, reconnection, or a host shared by multiple clients.
Metadata is available through the host and client for development and testing,
but is not shown in the terminal UI.

The native Swift UI is part of the project architecture but is not implemented
in v1.

## Planned technology

- Rust for the host and shared client library.
- [Ratatui](https://ratatui.rs/) for the first terminal UI.
- [UniFFI](https://mozilla.github.io/uniffi-rs/) to expose the shared Rust
  client to Swift.
- [XcodeGen](https://github.com/yonaskolb/XcodeGen) to generate and manage the
  native macOS Xcode project.
