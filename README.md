# File Peeker

File Peeker is a planned Finder-like file browser written in Rust. Its
filesystem logic can run locally at first and on another machine in the future.

This repository contains a minimal functional local browser. The shared client
starts and supervises a dedicated server over a private Unix socket, streams
directory entries, and is used by both Ratatui and SwiftUI frontends.

## Design

```text
Rust TUI  ─┐
           ├─> Shared client library -> Server process -> Filesystem
Swift UI ──┘
```

- The **server** runs on the machine being browsed and performs filesystem work.
- The **client** starts and connects to the server and exposes a UI-independent
  asynchronous file-browser interface.
- A **UI** handles input and display only. Rust/Ratatui and native macOS
  SwiftUI frontends are both available.

Separating these components allows the Rust TUI and Swift UI to share the same
client implementation and server behavior. Rust uses the client's native API;
Swift uses bindings generated by UniFFI. The native macOS project is generated
and managed with XcodeGen. The server protocol and process lifecycle remain
private to the client. The same boundary also allows a future remote connection
without moving filesystem logic into each UI.

The client produces filesystem results asynchronously but does not update or
render either UI. Ratatui forwards results into its application event loop;
SwiftUI applies them to observable state on the main actor.

Each `BrowserClient` owns one dedicated server process. A long-lived control
connection defines their shared lifetime, while each filesystem operation uses
its own short-lived connection. This permits independent operations without
request IDs or response routing on one socket.

## First version

V1 is a local, macOS-only, read-only browser. It:

- Start in a supplied path or the current directory.
- Stream and display the direct children of that directory.
- Include hidden entries and keep filesystem enumeration order.
- Allow entering child directories and directory symlinks.

V1 will not include parent navigation, sorting, previews, search, filesystem
changes, remote connections, reconnection, or a server shared by multiple clients.
Metadata is available through the server and client for development and testing,
but is not shown in the terminal UI.

The terminal UI opens an optional command-line path or the current directory.
The SwiftUI app opens the user's home directory.

## Technology

- Rust for the server and shared client library.
- [Ratatui](https://ratatui.rs/) for the first terminal UI.
- [UniFFI](https://mozilla.github.io/uniffi-rs/) to expose the shared Rust
  client to Swift.
- [XcodeGen](https://github.com/yonaskolb/XcodeGen) to generate and manage the
  native macOS Xcode project.

## Build and verify

Requirements are Rust, Swift/Xcode, and XcodeGen on macOS.

```text
make check          # Rust formatting, Clippy, and tests
make local-server-test
                    # Start the real local server and verify shutdown cleanup
make runnable-test  # Run a non-interactive local TUI listing smoke test
make bindings       # Build Rust and generate Swift bindings
make client-integration-test
                    # Compile, link, and test generated Swift types and an async call
make xcode-build    # Generate and compile the SwiftUI project
make verify         # Run the complete verification sequence
```

To exercise unpublished remote server installation through Cargo, use an SSH
target with Rust 1.88 or newer, Cargo, `tar`, and crates.io access:

```text
scripts/test-remote-server-install.sh SSH_DESTINATION
```

The server command line is parsed with Clap. Use
`file-peeker-server --help` for generated usage information,
`file-peeker-server --version` for its package version, and
`file-peeker-server version --format json` for machine-readable server and
protocol versions.

The script packages the unpublished server and protocol crates, transfers both
archives to the target, and runs `cargo install --path` against the packaged
server source with a test-only patch to the packaged protocol source. This is a
package-install smoke test; production installation will use crates.io. The
temporary remote directory is removed afterward.

The client also contains a private, non-exported installation helper exercised
by its Rust tests. It installs published server versions below
`~/.file-peeker/servers/<version>` and is not currently callable by either UI.

Generated Swift bindings, native build products, DerivedData, and the generated
`.xcodeproj` are build artifacts and are not committed.

The documents specify the intended functional behavior beyond the empty
skeleton.
